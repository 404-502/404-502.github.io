配置SQL server 2008远程连接

2023/9/9 文章 SQL server

一、SQL server 2008的配置

在任务栏的搜索框里搜索 服务 ，并打开

把输入法切换到英文，按 S 快速检索首字母为 S 的名称，然后继续找名字为 SQL Server Browser 右键属性，把禁用改为自动，点一下应用

再点一下启动，然后点击确定

在开始菜单栏打开SQL server配置管理器

按顺序点击启用

然后再右键点击属性，把已启用的否改为是

再点击IP地址，看哪个是本机IP地址，然后把对应的TCP动态端口0散掉，TCP端口填上1433，再把对应的已启用的否改为是

(若没有看到本机IP地址，就把IP1的地址改为实际的地址，后面就按上面的步骤继续操作)

滚到下面把IPAll里的两个端口分别改为3064和1433，保存

回到SQL server 2008管理器，右键选择重新启动

后面的是检查，如果没有改过默认配置，就可以跳过直接看防火墙的配置

分别检查这两个客户端协议里面的TCP/IP的属性

是否端口都是1433，和是否启用

如果更改了，和上面一样重新启动一下

打开SQL server 2008管理器，并登录进去，右键属性

点击连接，看是否打开允许远程连接

二、Windows防火墙配置

（若是低版本Windows server，就在开始菜单栏，打开管理工具，在里面找到高级安全 Windows Defender 防火墙打开）

在任务栏搜索框内搜索 高级安全 ，点击打开高级安全 Windows Defender 防火墙

下面两个方案选一个就可以了！！！

方案一

直接关闭防火墙（不推荐因为不安全，但对于教学环境比较省事）

点击防火墙属性

分别把上面三个配置文件的启用改为关闭，再点击确定就ok了

方案二

先点击入站规则，再点击新建规则

选择端口，然后下一步

如图所示，填端口1433，然后下一步

操作和配置文件保持默认即可，最后起个名字，完成

三、远程连接

打开连接的界面，填写服务器名称、登录名、密码

服务器名称填IP地址

或者点击服务器名称右边的小三角，选择浏览更多，点击网络服务器，等他搜索，然后选择确定，输入密码连接
